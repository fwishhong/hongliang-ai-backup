# 软件开发标准流程 (老洪制定)

> 日期: 2026-03-17

## 完整开发流程

### 阶段一：需求定义
1. **和 Claude 来回对话确定需求**，先将需求文档定稿

### 阶段二：竞品分析
2. **需求文档定好之后**，让 Claude 找网络相关的项目进行对比，找出不足

### 阶段三：技术方案
3. **功能完善后**，让 Claude 给出技术实现文档
4. **将技术实现文档和需求文档放到项目文件夹**

### 阶段四：Codex 开发
5. **进入项目文件夹打开 Codex**
6. **让 Codex 读取需求文档和技术文档**，分成多个开发周期，并给出对应周期的开发文档
7. **Codex 按开发周期开始开发**
8. **继续，一直继续**

### 阶段五：文档完善
9. **Codex 完成后**，让 Codex 根据项目结构和代码写一个完整流程功能文档

### 阶段六：优化迭代 (循环)
10. **将功能文档给 Claude**，让他评估和原始文档差别，给出优化清单
11. **Codex 按清单优化**
12. **优化完后**，让 Codex 更新功能流程文档
13. **再给 Claude**，重复这个流程，直到没有优化点

### 阶段七：测试验证 (循环)
14. **开始功能测试**，让 Claude 给出测试功能可靠性和完整性的清单和流程
15. **Codex 按清单排查**，输出给 Claude
16. **Claude 继续给测试清单和流程**，重复这个流程直到完成

---

## 核心原则
- **需求 → 技术 → 开发 → 文档 → 优化 → 测试**
- 循环迭代，直到完美
- 文档全程跟进
